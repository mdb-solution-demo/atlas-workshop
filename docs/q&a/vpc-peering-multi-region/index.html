1. Multi-Region VPC Peering

GCP VPC 是全局资源，VPC内部的子网作为分区资源，流量自动跨区域路由。而AWS VPC是区域资源:必须添加额外的资源来路由区域之间的流量。

所以Atlas和GCP打通VPC是全局VPC Peering(不需要指定具体Region)， GCP VPC架构如下图所示：

1.1 Atlas GCP Multi-Region Deployment with VPC Peering​

让我们在 MongoDB Atlas 中部署一个GCP多云集群，在开始之前，请确保您已经登录cloud.mongodb.com到您的账户。

  1. 点击 Build a Database 按钮，选择Google Cloud，Enable Atlas多云多区域功能，然后分别在GCP的 us-east1 和 us-west4创建多区域集群，如下图所示.

  2. 打通GCP和Atlas VPC Peering，具体过程不在这里阐述，结果如下：

  1. Atlas Overview中可以看到刚刚建立的多区域集群。

  2. 登录一台GCP VM，分别Ping两个Region的MongoDB Server域名，可以看到：全部都是通过内网地址连接，唯一区别是两个所在Region的子网不同
